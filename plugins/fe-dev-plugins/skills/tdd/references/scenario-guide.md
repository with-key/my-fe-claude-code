# 시나리오 작성 가이드

## 시나리오 분류 기준

요구사항을 분석하여 다음 테스트 파일 단위로 시나리오를 분류한다.
각 테스트 파일은 하나의 관심사만 검증한다.

| 테스트 대상 | 파일 패턴 | 검증 내용 |
|------------|----------|----------|
| 컴포넌트 렌더링/인터랙션 | `*.test.tsx` | UI 표시, 사용자 인터랙션 후 화면 변화 |
| 스키마/유효성 검증 | `*.test.ts` | 검증 규칙, 에러 메시지, 경계값 |
| 커스텀 훅 | `*.test.ts` | 상태 변화, 반환값, 비동기 흐름 |
| 유틸/헬퍼 | `*.test.ts` | 순수 함수의 입력 → 출력 |

## 시나리오 작성 규칙

- `describe`와 `it` 이름을 유저 스토리로 서술한다. 구현 세부사항(함수명, 상태명)을 넣지 않는다.
- 컴포넌트 테스트에서는 에러 메시지의 구체적 텍스트가 아닌, 에러 영역의 표시 여부만 시나리오에 포함한다.
- 정상 플로우, 경계값, 에러 케이스, 엣지 케이스를 포함한다.

## 산출물 형식

```markdown
# 테스트 시나리오: <기능명>

## <파일경로>.test.tsx
describe('<컴포넌트명>')
  it('<초기 렌더링 검증>')
  it('<사용자 인터랙션 검증>')
  describe('<조건>')
    it('<조건별 동작 검증>')

## <파일경로>.test.ts
describe('<스키마/훅/유틸명>')
  it('<입력 → 출력 검증>')
  it('<경계값 검증>')
  it('<에러 케이스 검증>')
```

## 주의사항

- 시나리오 단계에서는 **테스트 코드를 작성하지 않는다.** 시나리오 문서만 산출한다.
- 기존 프로젝트 코드를 탐색하여 테스트 파일 위치와 네이밍 패턴을 따른다.
