# 리팩토링 체크리스트

## 구현 코드 리팩토링 대상

- [ ] 중복 코드 제거
- [ ] 네이밍 개선 (변수, 함수, 컴포넌트)
- [ ] 함수/컴포넌트 분리 (단일 책임 원칙)
- [ ] 타입 안전성 강화 (`any` 제거, 적절한 타입 정의)
- [ ] 프로젝트 컨벤션 준수 (CLAUDE.md 참조)

## 테스트 코드 리팩토링 대상

- [ ] 중복된 setup 코드를 `beforeEach`로 추출
- [ ] 테스트 헬퍼/팩토리 함수 분리
- [ ] 불필요한 assertion 제거
- [ ] 테스트 이름의 명확성 개선

## 리팩토링 원칙

- **기능을 추가하지 않는다.** 외부 동작을 변경하지 않는 내부 구조 개선만 수행한다.
- 리팩토링 전후로 테스트를 실행하여 모든 테스트가 여전히 PASS인지 확인한다.
- 테스트가 하나라도 FAIL하면 리팩토링을 되돌리고 더 작은 단위로 나누어 재시도한다.
