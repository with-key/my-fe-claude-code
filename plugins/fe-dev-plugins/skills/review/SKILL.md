# /review — 코드 리뷰

> 트리거 키워드: 리뷰, review, 코드 검사, 품질 검사, 보안 검사

## 사용법

사용자가 전달한 액션: `$ARGUMENTS`

### 액션별 동작

**`/review`** — 기본 코드 리뷰
1. git diff로 현재 변경사항을 파악한다.
2. 변경된 파일을 **테스트 코드**와 **구현 코드**로 분류한다.
3. 각 카테고리별 리뷰 기준에 따라 시니어 프론트엔드 개발자 관점에서 검사한다.
4. Critical / Warning / Info 등급으로 결과를 출력한다.

**`/review --security`** — 보안 집중 리뷰
1. git diff로 현재 변경사항을 파악한다.
2. XSS, 인젝션, 인증 누락, 민감 데이터 노출을 중점 검사한다.
3. 보안 관련 문제를 등급별로 출력한다.

## 리뷰 기준

### 구현 코드 리뷰 (`*.ts`, `*.tsx` — 테스트 파일 제외)

**컨벤션 준수**
- 프로젝트 CLAUDE.md에 정의된 코딩 컨벤션을 따르고 있는가
- 네이밍 규칙(파일명, 변수, 함수, 컴포넌트)이 일관되는가
- 디렉토리 구조와 파일 배치가 프로젝트 패턴과 일치하는가

**디자인 시스템 준수**
- Base UI 컴포넌트를 사용하고 있는가 (불필요한 커스텀 컴포넌트를 만들지 않았는가)
- Panda CSS 디자인 토큰을 사용하고 있는가 (하드코딩된 색상, 간격, 폰트 크기가 없는가)
- 디자인 시스템에 정의된 패턴을 따르고 있는가

**웹 접근성**
- 시맨틱 HTML 요소를 적절히 사용하고 있는가
- aria 속성이 필요한 곳에 올바르게 적용되어 있는가
- 키보드 네비게이션이 가능한가
- 색상 대비가 충분한가

**Testability**
- 컴포넌트가 테스트하기 쉬운 구조로 작성되었는가
- 비즈니스 로직과 UI가 적절히 분리되어 있는가
- 외부 의존성이 주입 가능한 형태인가
- 테스트에서 쿼리할 수 있는 접근 가능한 요소(role, label 등)가 존재하는가

**코드 품질**
- 타입 안전성이 확보되었는가 (`any`, 타입 단언 남용이 없는가)
- 에러 처리가 적절한가 (API 호출, 비동기 처리)
- 불필요한 리렌더링을 유발하는 코드가 없는가

### 테스트 코드 리뷰 (`*.test.ts`, `*.test.tsx`, `*.spec.ts`)

**컨벤션 준수**
- BDD 스타일(`describe` → `it`)로 구조화되어 있는가
- 테스트 이름이 사용자 행동 → 기대 결과 형태로 서술되어 있는가
- `describe` 중첩으로 기능/컨텍스트가 명확히 그룹화되어 있는가

**테스트 유효성**
- 의미 없는 테스트가 아닌가 (항상 통과하는 테스트, 구현을 그대로 복사한 테스트)
- assertion이 충분하고 정확한가 (단순히 에러 없이 렌더링되는지만 확인하는 것은 부족)
- 경계값, 에러 케이스, 엣지 케이스를 커버하고 있는가

**테스트 전략 적절성**
- 유닛 테스트(`*.test.ts`)와 통합 테스트(`*.test.tsx`)가 적절히 구분되어 있는가
- 모키스트/클래시스트 전략이 맥락에 맞게 적용되었는가
  - 컴포넌트 자체의 렌더링과 인터랙션 → 클래시스트 (실제 구현으로 테스트)
  - 외부 시스템(API, 라우터, 브라우저 API) → 모키스트 (mock/msw 사용)
- 불필요한 mock이 없는가 (내부 모듈이나 자식 컴포넌트를 과도하게 mock하지 않았는가)
- API mock은 `msw`를 우선 사용하고 있는가

**테스트 품질**
- 테스트 간 의존성이 없는가 (실행 순서에 의존하지 않는가)
- setup/teardown이 적절히 관리되고 있는가
- React Testing Library의 쿼리 우선순위를 따르고 있는가 (getByRole > getByLabelText > getByText > getByTestId)

## 출력 형식

```
┌─ Code Review ───────────────────────────┐
│ Scope:   <리뷰 범위>                     │
│ Files:   <변경 파일 수>                   │
│ Issues:  <발견된 문제 수>                 │
│ Quality: ● Good / ▲ Needs Work / ✗ Poor │
├──────────────────────────────────────────┤
│ 구현 코드: <파일 수>개  테스트 코드: <파일 수>개 │
└──────────────────────────────────────────┘
```

이슈 출력 시 구현 코드와 테스트 코드를 섹션으로 구분하여 표시한다.

## 주의사항
- 코드를 수정하지 않는다. 리뷰 결과만 출력한다.
- git 저장소가 아닌 환경에서는 사용할 수 없다.
- CLAUDE.md가 있으면 프로젝트 컨벤션 기준으로, 없으면 일반적인 프론트엔드 베스트 프랙티스 기준으로 리뷰한다.
