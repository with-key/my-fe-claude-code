# /impact — 변경 영향 범위 빠른 분석

> 트리거 키워드: 영향 분석, impact, PRD 변경, 변경 범위, 기획 변경

## 사용법

사용자가 전달한 액션: `$ARGUMENTS`

### 액션별 동작

**`/impact <PRD 경로>`** — 단일 PRD 기반 영향 분석
1. 지정된 PRD 파일을 읽어 요구사항을 파악한다.
2. PRD의 핵심 키워드로 코드베이스를 검색한다.
3. 영향받는 파일을 수정 필요(●)와 확인 필요(○)로 분류한다.
4. 각 파일에 대한 한 줄 수정 힌트를 제공한다.

**`/impact <변경 설명>`** — 자유 텍스트 기반 영향 분석
1. 변경 설명에서 핵심 키워드를 추출한다.
2. 키워드로 코드베이스를 검색한다.
3. 영향받는 파일을 분류하고 한 줄 수정 힌트를 제공한다.

**`/impact diff <이전 PRD> <이후 PRD>`** — 두 버전 비교 영향 분석
1. 두 PRD 파일의 차이점을 분석한다.
2. 변경된 요구사항에 영향받는 코드를 검색한다.
3. 변경 전후의 차이와 한 줄 수정 힌트를 제공한다.

## 출력 형식

```
┌─ Impact Analysis ───────────────────────┐
│ PRD Change: <변경 요약>                   │
│ Affected:                                │
│   ● <파일경로>    (<수정 이유>)            │
│   ○ <파일경로>    (<확인 이유>)            │
│                                          │
│ ● 수정 필요  ○ 확인 필요                  │
└──────────────────────────────────────────┘

## 수정 힌트

### ● <파일경로>
- <한 줄 수정 힌트>

### ○ <파일경로>
- <확인이 필요한 이유>

---
> 상세 수정 방안은 `/fix`를 사용하세요.
```

## 주의사항
- 코드를 수정하지 않습니다. 분석 결과만 출력합니다.
- 확실하지 않은 영향은 "확인 필요(○)"로 분류합니다.
- 수정 방법의 상세 안내는 `/fix`의 역할입니다. 여기서는 "무엇이 영향받는가"에 집중합니다.
